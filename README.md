# SQLAlchemy Mastery: Sync vs Async & Core vs ORM

Учебный проект для глубокого изучения и тестирования возможностей **SQLAlchemy 2.0**. Основная цель — сравнение подходов к проектированию БД и выполнению запросов в различных режимах.

## 🚀 Основные цели
- Сравнение производительности и синтаксиса **Core** (выражения через `text`, `insert`, `select`) и **ORM** (работа через модели `Mapped`).
- Переход от **синхронного** взаимодействия к **асинхронному** (`asyncpg`).
- Изучение паттернов управления сессиями (`sessionmaker`, `AsyncSession`).

## 🛠 Технологический стек
* **Python 3.10+**
* **SQLAlchemy 2.0+** (Declarative Mapping)
* **PostgreSQL** (через `psycopg2` и `asyncpg`)
* **Pydantic Settings** (валидация конфигурации через `.env`)

Репозиторий содержит материалы по изучению **SQLAlchemy 2.0**, а также выполнение лабораторных работ по базам данных. Проект разделен на две основные части: практическое тестирование различных подходов SQLAlchemy и выполнение академических заданий.

## 📂 Структура проекта

### 1. Project 1: SQLAlchemy Deep Dive
Основной модуль для тестирования и сравнения современных подходов работы с БД.
* `queries/core.py` — реализация запросов с использованием **SQLAlchemy Core** (SQL Expression Language).
* `queries/orm.py` — реализация запросов через **ORM** (Object-Relational Mapping).
* `database.py` & `models.py` — конфигурация движков (Engine) и описание моделей данных.
* `config.py` — управление настройками через `.env`.

### 2. Homework_bd_10
Лабораторная работа для университета.
* `database/` — настройки подключения и модели для конкретной задачи.
* `queries/` — набор скриптов для управления данными:
    * `create_tables.py` — инициализация структуры БД.
    * `insert_data.py` — наполнение таблиц тестовыми данными.
    * `queries.py` — выполнение целевых запросов согласно заданию.
